# CLAUDE.md

## プラン作成・実装方針

- プランをフェーズに分けない。「目指すべき最終形」を 1 本のプランとして提示し、ユーザー確認後にすべて実装してから動作確認する
- フェーズごとの動作維持のために旧実装を温存したり、feature flag / deprecated alias / 互換 shim を残したりしない。最終形に不要なコードは初手から書かない (後で削除する前提のコードを書かない)
- 既存コードの一部を残す合理的理由 (migration コスト・外部互換) が無い限り、置き換え対象は完全削除して新コードでクリーンに再構築する
- 実装が一通り終わってからまとめて `/tidy` → `/docs` → `/commit` を実行する (フェーズごとには実行しない)
