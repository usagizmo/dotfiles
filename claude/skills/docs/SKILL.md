# ドキュメント更新チェック

次回以降のセッションでエージェントがより正確に動けるよう、ドキュメントを最新化する。
加えて、セッション全体を俯瞰した設計振り返りを行い、「次回から指示なしに美しい設計を選べる」ための原則を蒸留・永続化する。
実装者の「自分はわかっているから不要」というバイアスを排除するため、サブエージェントに委譲して第三者視点で評価する。

## 対象ドキュメント

| 種類 | スコープ | 何を書くか |
| --- | --- | --- |
| **CLAUDE.md** | 常時読み込み | プロジェクト概要・コマンド・スキル一覧 |
| **Rules** (`.claude/rules/`) | 常時読み込み | 全セッションで適用される規約・原則 |
| **Skills** (`.claude/skills/`) | 関連時に読み込み | 特定タスク用の手順・判断基準・リファレンス |

※ グローバル（`~/.claude/`）とプロジェクト（`.claude/`）の両方が対象。

## 実行手順

1. 今回のセッションで変更したファイルパスを把握する
2. セッション中の知見を整理する:
   - ハマりポイント・試行錯誤の末に見つけた正解パターン・設計判断の理由
   - セッション全体を通じて行った設計上の選択とその根拠
3. Agent ツールで `general-purpose` サブエージェントを起動し、この skill の `references/review-prompt.md` のテンプレートに以下を埋めて渡す:
   - `{変更したファイルパスの一覧}`
   - `{ハマりポイント・正解パターン・設計判断の理由}`
   - `{設計上の選択とその根拠}`
4. サブエージェントの結果に基づき、ドキュメントを更新する
5. 更新した箇所を簡潔に報告する
